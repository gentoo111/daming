# 大明帝国官网 - 界面设计文档

## 设计理念

**"现代政府官网的古代版本"** - 以明朝六部制为架构，打造古色古香的官方门户网站

- 🏛️ **官方权威感** - 像政府官网一样严谨规范
- 🎨 **古典美学** - 传统色彩、古典字体、对称布局
- 📱 **现代功能** - 保证用户体验和响应式设计
- 🔄 **功能映射** - 古代官职对应现代网站功能

## 网站整体架构

### 主导航结构
```
大明帝国官网
├── 首页 (紫禁城门户)
├── 六部衙门
│   ├── 吏部 (人事管理)
│   ├── 户部 (财政民生) 
│   ├── 礼部 (文化教育)
│   ├── 兵部 (游戏竞技)
│   ├── 刑部 (规则管理)
│   └── 工部 (技术建设)
├── 地方机构
│   ├── 府衙 (地区社群)
│   ├── 县衙 (小组织)
│   └── 驿站 (消息中心)
└── 个人空间 (府邸)
```

- **配色方案**: 故宫红 (#8B0000) + 金黄 (#FFD700) + 墨绿 (#2F4F2F) + 象牙白 (#FFFFF0)
- **装饰元素**: 云纹、龙纹、回文边框、印章元素
- **字体**: 思源宋体/方正字体 + 英文用衬线字体
- **图标**: 古典印章风格的功能图标

## 六部详细设计

### 🏛️ 吏部 - 人事管理衙门

**页面标题**: "吏部衙门 - 天下英才皆入彀中"

**功能模块**:
- **📜 护照申请司** (用户注册)
  - "申请大明子民身份证明"
  - 表单：姓名、字号、籍贯、出身
  - 生成古风身份证件
  
- **👥 官职查询司** (用户等级系统)
  - 官职等级：白身→生员→举人→进士→翰林→尚书
  - 个人履历展示
  - 升迁记录查询

- **📋 科举报名司** (考试/竞赛系统)
  - 乡试、会试、殿试报名
  - 考试安排和成绩查询
  - 金榜题名展示

- **📊 官员名册** (用户列表/排行榜)
  - 当朝文武百官
  - 按地区、按官职分类
  - 德望排行榜

**界面特色**: 用印章、竹简、文书等元素装饰

### 💰 户部 - 财政民生衙门

**页面标题**: "户部衙门 - 国富民强，仓廪实而知礼节"

**功能模块**:
- **💎 国库司** (虚拟经济中心)
  - 个人银两查询
  - 收入支出记录
  - 投资理财工具

- **📈 商贸司** (虚拟股市/交易)
  - 丝绸之路贸易模拟
  - 古代版股票交易
  - 商会投资项目

- **🌾 田赋司** (积分/奖励系统)  
  - 日常任务完成获取"税收"
  - 签到奖励
  - 活动奖励发放

- **📊 户籍司** (数据统计)
  - 帝国人口统计
  - 各地经济数据
  - 用户活跃度统计

**界面特色**: 算盘、铜钱、账本、丝绸等商贸元素

## 个人空间 - 数字府邸

### 设计概念
仿照古代府邸布局，分为不同功能区域：

```
个人府邸布局:
┌─────────┐
│   正厅   │ <- 个人信息展示
├─────────┤
│   书房   │ <- 学习记录/成就
├─────────┤  
│   库房   │ <- 道具收藏
├─────────┤
│   花园   │ <- 社交空间
└─────────┘
```

### 功能区域
- **正厅**: 个人信息、官职展示、访客留言
- **书房**: 学习记录、证书展示、藏书收藏
- **库房**: 道具装备、收藏品、财富统计  
- **花园**: 好友列表、动态分享、互动空间

## 响应式设计要求

### 桌面端 (1200px+)
- 完整的六部导航
- 丰富的装饰元素
- 多列布局展示

### 平板端 (768px-1199px)
- 折叠式导航菜单
- 适度简化装饰
- 两列布局

### 移动端 (320px-767px)
- 汉堡包菜单
- 简化装饰元素
- 单列布局
- 底部导航栏

## 开发优先级

### 第一阶段：静态官网架构 (2周)
1. **首页布局** - 完整的视觉设计和导航
2. **六部入口页** - 各部门的介绍和功能概览
3. **基础样式系统** - 古风UI组件库
4. **响应式布局** - 适配各种设备

### 第二阶段：核心功能页面 (3周)
1. **吏部-护照申请** - 用户注册功能
2. **个人空间** - 基础的个人信息展示
3. **简单交互** - 基础的点击跳转功能
4. **数据展示** - 静态的统计数据展示

### 第三阶段：动态功能接入 (持续)
1. 用户系统集成
2. 数据库功能接入
3. 实际业务逻辑实现
4. 高级交互功能

## 技术实现要点

### CSS框架使用
- TailwindCSS + 自定义古风主题
- CSS Grid/Flexbox实现复杂布局
- CSS动画实现古典过渡效果

### 组件设计原则
- 可复用的古风UI组件
- 统一的色彩和字体规范
- 模块化的功能组件
- 易于维护的代码结构

### 性能优化
- 图片懒加载
- 组件按需加载
- CSS/JS压缩优化
- CDN资源分发
